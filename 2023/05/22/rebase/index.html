git rebase

之前一直使用pull来合代码，是merge操作，突发奇想想用rebase合代码结果出了些问题

建议看这个视频


merge: 会按照时间顺序去合并，但会新创建一个merge的commit


rebase: 重新创建所有的commit，且不会有rebase节点，你的commit会被放到最前面，主线上的commit都会被放在后后面,这也是所谓的变基
